9 Wege ein Bild zu speichern

Quelle: assets/foreground v3.png — teste jede Methode auf deinem Gerät.

Erkenne Umgebung…
01

Web Share API

navigator.share({ files })

Öffnet das native Share Sheet (iOS Safari 15+, Android Chrome). User wählt „Sichern in Fotos" / „Herunterladen".

✓ iOS Safari, Android Chrome, PWA · ✗ Meiste In-App Browser

02

Klassischer Anchor

<a download>

Der Klassiker. Funktioniert auf Desktop und Android Chrome zuverlässig — iOS Safari öffnet das Bild stattdessen inline.

✓ Desktop, Android Chrome · ✗ iOS, In-App Browser

03

Long-Press Modal

<img> + Drücken

Bild inline anzeigen — User hält das Bild gedrückt und nutzt das native Kontextmenü „Bild sichern".

✓ Universell · benötigt User-Aktion

04

Neuer Tab

window.open(blobUrl)

Bild in neuem Tab öffnen — User kann dort speichern. Popup-Blocker und In-App-Browser hindern dies oft.

✓ Desktop, Mobile-Browser · ✗ Viele In-App-Browser

05

Tab-Navigation

location.href = blobUrl

Aktuellen Tab zum Bild navigieren — funktioniert fast überall, zerstört aber den App-Zustand (Back-Button nötig).

✓ Universell · App-State geht verloren

06

Zwischenablage

clipboard.write(ClipboardItem)

Bild in die System-Zwischenablage kopieren — User fügt es direkt in Fotos, Notes oder Instagram ein.

✓ iOS 13.4+, Android Chrome · kein „Save", aber „Paste"

07

Service Worker

Content-Disposition: attachment

SW fängt /__download/* ab und sendet das Bild mit Attachment-Header — erzwingt echten Download wo <a download> ignoriert wird.

✓ iOS Safari, Android Chrome · ✗ Meiste In-App-Browser (kein SW)

08

QR Code

Scan with another device

QR Code für das Bild generieren — mit einem zweiten Gerät scannen und außerhalb des In-App-Browsers laden.

✓ Bulletproof · benötigt zweites Gerät

09

In externem Browser öffnen

Detect WebView + Hinweis

In-App-Browser erkennen und plattform-spezifische Anweisung zeigen, wie der User in Safari/Chrome wechselt.

✓ Universeller UX-Trick

Bild speichern

Halte das Bild gedrückt und wähle „Bild sichern".
